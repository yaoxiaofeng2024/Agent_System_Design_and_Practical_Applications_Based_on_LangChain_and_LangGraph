# 智能体实战之生产部署：从开发环境到生产级服务

## 一.简介

前面的章节聚焦于Agent的设计与实现，但一个Agent只有成功部署到生产环境并稳定运行，才能真正发挥价值。本章将补全从"能跑"到"能上线"的关键工程知识，这是企业级Agent开发中不可或缺的环节。

### 1. Agent部署的挑战

与传统Web应用不同，Agent部署面临独特挑战：

* **长耗时请求**：Agent可能需要多轮推理和工具调用，单次请求耗时从数秒到数分钟不等，传统HTTP请求模式会超时。
* **非确定性输出**：同样的输入可能产生不同输出，需要特殊的测试和监控策略。
* **Token成本控制**：LLM调用按Token计费，需要监控和控制成本。
* **流式输出**：用户期望实时看到Agent的思考过程和中间结果，而非等待最终答案。
* **并发与限流**：LLM API有速率限制，需要合理管理并发请求。

### 2. FastAPI服务化

FastAPI是构建Agent API服务的理想选择：

* **原生异步支持**：`async/await`语法天然适配LLM的异步调用。
* **自动API文档**：OpenAPI/Swagger文档自动生成，方便前后端对接。
* **请求验证**：Pydantic模型自动验证请求参数，减少错误。
* **SSE支持**：Server-Sent Events实现流式输出，无需WebSocket。
* **中间件生态**：CORS、认证、日志等中间件丰富。

### 3. 流式输出（Streaming）

流式输出是Agent用户体验的关键。三种主流实现方式：

* **Server-Sent Events（SSE）**：基于HTTP的单向流，服务端主动推送，实现简单，适合文本流。LangChain的`.stream()`和`.astream()`天然支持。
* **WebSocket**：双向通信，适合需要客户端也发送消息的场景（如中断Agent执行）。
* **Chunked Transfer**：HTTP分块传输，最简单但功能有限。

SSE是最推荐的Agent流式输出方案，因为：
- 基于标准HTTP，兼容性好，不需要特殊协议。
- 自动重连机制。
- 与LangChain的流式API完美配合。

### 4. 可观测性（Observability）

生产环境中的Agent必须具备可观测性，即能够回答：Agent在做什么？做得怎么样？出了什么问题？

三大支柱：

* **日志（Logging）**：记录Agent的每一步操作，包括输入、输出、耗时、工具调用等。
* **指标（Metrics）**：量化Agent的运行状态，如请求量、延迟、Token消耗、成功率等。
* **追踪（Tracing）**：记录一次请求的完整调用链路，包括每个节点的输入输出和耗时。

**LangSmith**是LangChain生态的可观测性平台，提供：
- 自动追踪LangChain/LangGraph的每一步执行。
- 可视化调用链路，快速定位问题。
- 支持A/B测试和评测集管理。
- Token消耗和成本追踪。

### 5. Docker容器化

Docker是Agent部署的标准方式：

* **环境一致性**：开发、测试、生产环境完全一致，消除"在我机器上能跑"的问题。
* **依赖隔离**：Python版本、系统库、模型文件等全部打包在镜像中。
* **弹性伸缩**：配合K8s实现自动扩缩容。
* **快速回滚**：镜像版本化管理，出问题秒级回滚。

### 6. 健康检查与优雅关闭

生产服务必须具备：

* **健康检查端点**：`/health`端点供负载均衡器检测服务状态。
* **优雅关闭**：收到终止信号时，等待正在处理的请求完成后再退出，避免中断用户请求。
* **就绪检查**：`/ready`端点检测服务是否已初始化完成（如模型加载完毕）。

### 1. 完整代码

```python
import time
import json
import asyncio
from contextlib import asynccontextmanager
from typing import Optional

from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import StreamingResponse
from pydantic import BaseModel, Field
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

from init_client import init_llm

llm = init_llm(0.1)

knowledge_base = {

                "花呗分期支持3期、6期、12期，3期免息，6期和12期收取手续费。订单金额需满100元才可使用花呗分期。",
    "会员权益": "普通会员：享受9.5折优惠。银卡会员（年消费满2000元）：享受9折优惠，每月1张免邮券。" +

class ChatRequest(BaseModel):
    question: str = Field(..., min_length=1, max_length=1000, description="用户问题")
    stream: bool = Field(default=False, description="是否使用流式输出")


class ChatResponse(BaseModel):
    question: str
    answer: str
    latency_ms: float


class HealthResponse(BaseModel):
    status: str
    model_loaded: bool
    uptime_seconds: float


start_time = time.time()


@asynccontextmanager
async def lifespan(app: FastAPI):
    print("🚀 服务启动中... 模型已加载")
    yield
    print("🛑 服务关闭中...")


app = FastAPI(
    title="客服Agent API",
    description="基于LangChain的电商客服智能问答服务",
    version="1.0.0",
    lifespan=lifespan
)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)


@app.get("/health", response_model=HealthResponse)
async def health_check():
    return HealthResponse(
        status="healthy",
        model_loaded=True,
        uptime_seconds=time.time() - start_time
    )


@app.get("/ready")
async def readiness_check():
    try:
        test_prompt = ChatPromptTemplate.from_template("回复OK")
        chain = test_prompt | llm | StrOutputParser()
        chain.invoke({})
        return {"ready": True}
    except Exception:
        raise HTTPException(status_code=503, detail="Model not ready")


@app.post("/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    global request_count, total_latency
    start = time.time()

    context = "\n\n".join([f"{k}：{v}" for k, v in knowledge_base.items()])
    prompt = ChatPromptTemplate.from_template(
        "你是一位专业的电商客服。请根据以下知识库信息回答用户问题。\n\n"
        "知识库：\n{context}\n\n"
        "用户问题：{question}\n\n"
        "要求：只基于知识库信息回答。如果知识库中没有相关信息，请明确说明。回答："
    )
    chain = prompt | llm | StrOutputParser()
    answer = chain.invoke({"context": context, "question": request.question})

    latency = (time.time() - start) * 1000
    request_count += 1
    total_latency += latency

    return ChatResponse(
        question=request.question,
        answer=answer,
        latency_ms=round(latency, 2)
    )


@app.post("/chat/stream")
async def chat_stream(request: ChatRequest):
    context = "\n\n".join([f"{k}：{v}" for k, v in knowledge_base.items()])
    prompt = ChatPromptTemplate.from_template(
        "你是一位专业的电商客服。请根据以下知识库信息回答用户问题。\n\n"
        "知识库：\n{context}\n\n"
        "用户问题：{question}\n\n"
        "要求：只基于知识库信息回答。如果知识库中没有相关信息，请明确说明。回答："
    )
    chain = prompt | llm | StrOutputParser()

    async def event_generator():
        async for chunk in chain.astream({"context": context, "question": request.question}):
            yield f"data: {json.dumps({'content': chunk}, ensure_ascii=False)}\n\n"
        yield f"data: {json.dumps({'done': True})}\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"}
    )


@app.get("/metrics")
async def metrics():
    avg_latency = total_latency / request_count if request_count > 0 else 0
    return {
        "total_requests": request_count,
        "average_latency_ms": round(avg_latency, 2),
        "uptime_seconds": round(time.time() - start_time, 2)
    }


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

## 五.关键知识点总结

### Agent部署架构选择

| 架构 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 单机FastAPI | 小规模、低并发 | 简单、成本低 | 单点故障、无法扩展 |
| Docker+K8s | 中大规模、高可用 | 弹性伸缩、滚动更新 | 运维复杂度高 |
| Serverless | 间歇性请求 | 按需付费、零运维 | 冷启动延迟、有状态难处理 |
| LangGraph Platform | LangGraph项目 | 内置状态管理、流式支持 | 需付费（Cloud版） |

### 流式输出方案对比

| 方案 | 方向 | 复杂度 | 兼容性 | 适用场景 |
|------|------|--------|--------|---------|
| SSE | 单向 | 低 | 好 | 文本流式输出（推荐） |
| WebSocket | 双向 | 中 | 好 | 需要客户端中断Agent |
| Chunked Transfer | 单向 | 低 | 一般 | 最简单的流式方案 |
